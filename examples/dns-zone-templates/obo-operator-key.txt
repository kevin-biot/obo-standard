; OBO DNS Anchoring Profile - Operator Signing Key
; draft-obo-agentic-evidence-envelope-01 Appendix E.3
;
; Publish the operator's Ed25519 signing key in DNS.
; The domain component of the issuer_id in the OBO Credential resolves to this record.
;
; Replace:
;   <operator-domain>      - your operator domain (e.g. ai.example.com)
;   <base64url-public-key> - base64url-encoded raw Ed25519 public key (no padding)
;
; DNSSEC signing is REQUIRED for regulated lane use.
; Recommended TTL: 300 seconds (5 minutes) for production key material.

_obo-key.<operator-domain>  300  IN  TXT  "v=obo1 ed25519=<base64url-public-key>"

; Example (replace with real key):
; _obo-key.ai.example.com  300  IN  TXT  "v=obo1 ed25519=AbCdEf..."

; Key rotation:
; 1. Generate new keypair
; 2. Publish the new value at _obo-key.<operator-domain>
; 3. Issue new credentials signed with new key, referencing new record
; 4. After all old credentials expire, remove old record
; Old credentials remain verifiable until their expires_at passes.
